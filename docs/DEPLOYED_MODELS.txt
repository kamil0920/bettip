Deployment Summary — Calibration Fix Retraining (2026-03-15 v11)
================================================================

CALIBRATION FIX (commit c99f1ac): All niche markets retrained with 80/20 calibration split.
Previous models had sigmoid calibration overfit (|a|=20-40, should be <10). Root cause: CalibratedClassifierCV
with cv="prefit" was fitted on same data as base model, producing degenerate steep sigmoids.
Fix: train on first 80%, calibrate on held-out last 20%. LGB/XGB now have |a|=0.1-3.0 (correct).
CatBoost inherently produces confident predictions → |a|=5-28 even on held-out data, caught by
production safety net (|a|>10 → falls back to raw base estimator).

Additional safety: niche market probs capped at 0.95 in generate_daily_recommendations.py.

Training: 700 Optuna trials, adversarial 2,10,0.75, mrmr=30, max_rfe=60, no_fastai, holdout_folds=2.
All models from 5 CI waves (runs 23107344654-23107661075), artifacts downloaded and uploaded to HF Hub.

LEGEND:
  Prec = Optimization set precision (primary ranking metric)
  HO_P = Holdout precision (out-of-sample, most trustworthy)
  HO_B = Holdout bets (sample size)
  ECE  = Holdout Expected Calibration Error (<0.05 = excellent, <0.10 = acceptable)
  FVA  = Forecast Value Added vs market baseline (higher = more skill)
  #F   = Number of features
  Odds = REAL (bookmaker) or EST (Poisson-estimated, ROI unreliable)
  20?  = Passes min_bets=20 gate (Y/N)

NOTE: ROI omitted for EST-odds markets (inflated by circular Poisson loop).
